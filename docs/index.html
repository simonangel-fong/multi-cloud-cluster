01 — Challenge

Kubernetes Multicloud

The practice of deploying and managing Kubernetes clusters across multiple cloud providers simultaneously — and making it operationally sustainable.

Why multi-cloud?
  • Resilience — survive a regional or provider-level outage
  • Vendor flexibility — avoid lock-in, negotiate from a stronger position
  • Workload placement — put each workload where it is cheapest or closest to data
  • Cost optimization — combine each provider's savings plans
Trade-offs
  • Operational overhead — multiple control planes, IAM models, networking stacks
  • Cost of consistency — keeping clusters, manifests, and observability in sync requires extra tooling and discipline

Real-world examples

# Cost relief from lock-in
Soaring infrastructure costs from a single cloud
can be reduced by combining each provider's
savings plans in a multicloud setup.
# Compliance & operational flexibility
Microsoft 365-integrated workloads → Azure
High-performance batch jobs → AWS
Each cloud where it is strongest.

02 — Architecture

System Overview

A single ArgoCD ApplicationSet fans out to both EKS and AKS, selecting cloud-specific Helm values per cluster. Cloudflare sits in front of both for load balancing. Grafana Cloud collects metrics from both clusters via Alloy.

Cloud Providers

AWS EKS & Azure AKS as target clusters

GitOps

ArgoCD + Helm for declarative, automated deployments

Observability

Grafana Cloud + Grafana Alloy for unified monitoring

Traffic Control

Cloudflare Load Balancer in front of both clusters

03 — Features

What This Project Demonstrates

Three end-to-end capabilities, each built on real tooling with production-grade patterns.

Multi-cloud Deploy

A single ArgoCD ApplicationSet fans out to both EKS and AKS using the clusters generator. Cloud-specific Helm values are selected per cluster via the cloud label.

Traffic Control

Cloudflare Load Balancer routes traffic across both clusters with health checks, failover, and geo-routing — giving resilience at the DNS layer without touching the application.

Unified Observability

Grafana Alloy scrapes metrics and logs from both clusters and ships them to a single Grafana Cloud workspace — one dashboard, two clouds.

ApplicationSet — key excerpt

apiVersion: argoproj.io/v1alpha1
kind: ApplicationSet
spec:
  generators:
    - clusters:
        selector:
          matchLabels:
            workload: demo-api
  template:
    spec:
      source:
        helm:
          valueFiles:
            - 'values-{{ index .metadata.labels "cloud" }}.yaml'
      destination:
        server: "{{ .server }}"
