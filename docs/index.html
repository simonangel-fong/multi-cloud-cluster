Multi-cloud Cluster

One app. Two clouds. GitOps end to end.

A project that runs a single application across AWS EKS and Azure AKS with shared observability and traffic control.

View on GitHub See How It Works

Business Challenge & Solution

Relying on a single cloud provider is a hidden business risk, because:

  • A single provider outage takes the entire service offline with no fallback and no warning
  • Pricing increases or contract changes leave the business with no negotiating power
  • Migrating to another provider later means months of costly re-engineering

How do businesses protect themselves from cloud dependency?

This project solves the problem by:

  • Creating multi-cloud clusters on AWS EKS and Azure AKS using Terraform
  • Deploying an application across both clusters simultaneously
  • Applying GitOps practices to automate and standardise deployment

Benefits

  • Resilience Survive a regional or provider-level outage without service interruption.
  • Cost control Combine savings plans across providers and avoid lock-in pricing.
  • Flexibility Place each workload on the cloud where it performs best.

Trade-offs

  • More moving parts Multiple control planes and networking stacks to manage across providers.
  • Consistency overhead Keeping both environments in sync requires extra tooling and discipline.

Feature: Multi-cloud Deploy

How it works

  1. Register the clusters AWS EKS and Azure AKS are registered in ArgoCD, each labelled with its cloud provider
  2. One config, two targets A single ArgoCD ApplicationSet reads those labels and deploys to every matching cluster automatically
  3. Cloud-specific settings Each cluster receives its own Helm values, so cloud differences are handled without duplicating code

Feature: Traffic Control

  • A Cloudflare Load Balancer sits in front of both clusters and steers traffic between the AWS and Azure pools
  • Adjusting the pool weights shifts load between clouds without touching DNS or the clusters themselves, which is useful for cost control, failover drills, and gradual cutovers

Feature: Cloud-agnostic Monitoring

Both clusters ship metrics and logs to a single Grafana Cloud workspace, making monitoring cloud-agnostic.

  • The demo API exports Prometheus metrics
  • Grafana Alloy runs in each cluster and forwards metrics and logs to Grafana Cloud
  • Grafana Cloud dashboards render data from both clusters side by side
